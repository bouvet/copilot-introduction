# Exercise 04 (Power BI) - GitHub.com Features

**Time:** ~10 minutes  
**Goal:** Use GitHub Copilot on GitHub.com for a DAX bugfix PR.

---

## Background

In this version, you fix a bug in a DAX measure file and then use Copilot to generate a pull request description.

---

## Steps

### 1. Fix the bug

Open `bugfix.tmdl`.

Ask Copilot Chat:

- "Find any logic bugs in this measure and explain the business impact."

Fix the measure.

### 2. Commit changes

Use git to commit the bugfix file.

### 3. Push branch/fork

Push your branch/fork as you normally would.

### 4. Create pull request on GitHub.com

Open a PR and use the Copilot button in the description box.

### 5. Review generated PR text

Check whether the generated description explains:

- what was wrong
- what changed
- impact on report numbers

---

## Optional

Create an issue titled:

- "Gross Margin % appears inflated when slicers are cleared"

Ask Copilot in the issue for likely root causes in DAX.
